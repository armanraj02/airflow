# ✈️ Airflow DAGs Project

A collection of Apache Airflow DAGs demonstrating core Airflow concepts — parallel execution, branching, backfilling, incremental processing, and fan-in/fan-out pipelines — all running in a fully Dockerized environment.

---

## 📁 Project Structure

```
airflow/
├── dags/                   # All Airflow DAG definitions (Python files)
├── docker/
│   ├── docker-compose.yaml # Airflow environment setup
│   └── api/                # Simple Flask API used as an events endpoint
└── output/
    ├── images/             # Images downloaded by rocket launch DAGs
    └── data/               # Processed data from pipeline DAGs
```

## 🚀 Getting Started

### Prerequisites

Make sure you have the following installed before proceeding:

- [Docker](https://docs.docker.com/get-docker/) (v20.10+)
- [Docker Compose](https://docs.docker.com/compose/install/) (v2.0+)

### Installation & Setup

1. **Clone the repository:**
   ```bash
   git clone https://github.com/armanraj02/airflow.git
   cd airflow
   ```

2. **Navigate to the Docker directory:**
   ```bash
   cd airflow_docker
   ```

3. **Start the Airflow environment:**
   ```bash
   docker-compose up -d
   ```

4. **Access the Airflow UI:**

   Open your browser and go to → `http://localhost:8080`

   | Field    | Value   |
   |----------|---------|
   | Username | `admin` |
   | Password | `admin` |

> **Note:** The first startup may take a minute or two while Docker pulls the required images and initializes the database.

## 🗂️ Available DAGs

| DAG | Description |
|-----|-------------|
| `parallel_dag` | Downloads rocket launch data and images from the [TheSpaceDevs API](https://thespacedevs.com/) and runs sample tasks in parallel. |
| `download_rocket_launcher` | A simple sequential DAG that downloads rocket launch data and images from the TheSpaceDevs API. |
| `download_rocket_launcher_parallel` | A parallel version of the rocket launcher DAG — same data source, tasks run concurrently. |
| `data_processing` | Fetches events from the Flask API, calculates statistics, and supports **backfilling** for past dates. Scheduled to run daily. |
| `pre_incremental_data_processing` | Incrementally processes only the current day's events from the Flask API — a more efficient alternative to full reprocessing. |
| `03_branching_dags` | Uses `BranchPythonOperator` to fetch sales data from either an old or new ERP system depending on the execution date. |
| `fan_in_out` | A full fan-in/fan-out pipeline — fetches sales and weather data in parallel, cleans and joins the datasets, then trains and deploys a model. |
| `01_unscheduled` | A simple unscheduled DAG that fetches data and calculates basic statistics. Good for manual trigger testing. |

---

## 🌐 Flask Events API

The `docker/` directory includes a lightweight Flask API that acts as a mock events endpoint. It is used by the `data_processing` and `pre_incremental_data_processing` DAGs to simulate fetching real-world event data. The API is automatically started as part of the `docker-compose` setup — no additional configuration needed.

---

## 📤 Output

All output generated by the DAGs is stored in the `output/` directory:

- `output/images/` — Images downloaded by the rocket launch DAGs
- `output/data/` — Processed statistics and data from the pipeline DAGs
